Design notes — Property surface

  • NEWWho-owes person tag. The shipped RentalCard drops ownerPersonId entirely (Property.tsx:290), even though the data carries it and the input-tab row shows it (HousingPaymentsTab.tsx:107). The ideal surfaces a person/Joint pill so a two-person household can tell at a glance whose obligation it is.
  • NEWEdit affordance on the card. Shipped card only has a destructive "Remove" (Property.tsx:299); editing requires going to Inputs. Owned-property cards already have "Edit value", so a card-level edit keeps the row consistent.
  • NEWDedicated aggregate card. Shipped total is an inline muted line in the section header (Property.tsx:548-555). Promoting it to a slate-accented card mirrors the weight of the owned-property "Equity" figure and reads as a first-class number.
  • NEW"Add rental" on-page. Shipped page has no add button — the empty state and all creation route to Inputs. A dashed ghost card lets users add without leaving the page (the affordance can deep-link to the Inputs tab to reuse the existing form, so no new form is required).
  • Status chip ("ongoing" / "ends in N mo"). Derived from endDate via the existing isActiveOn helper; turns the bare date into a glanceable state.
  • Kept as-is: the grid-cols-1 md:grid-cols-3 gap-4 rhythm, 8px radius, tabular-nums figures, and "From / Until" date pair all match the shipped card and the 3-card convention.

Design notes — Vehicles surface

  • Structurally identical to the Property surface — same deltas apply (Vehicles.tsx:281 LeaseCard mirrors RentalCard and shares the same omissions). Fixing one component pattern fixes both.
  • Term-remaining chip ("21 mo left"). Leases are almost always fixed-term, so months-remaining is more useful here than the open-ended "ongoing" common to rent.
  • Aggregate copy reads "Total recurring vehicle" to parallel "Total recurring housing"; shipped label is the generic "Total monthly obligation" (Vehicles.tsx:541).

Design notes — Spending tile

  • NEWRent vs lease split. Shipped tile shows only a combined figure plus a "N rentals, M leases" count line (Spending.tsx:247-253). Splitting the dollar amounts (using the already-exported monthlyHousingObligation / monthlyLeaseObligation helpers) makes the tile answer "which is driving this?" without a click.
  • Card styling parity. The shipped tile uses a one-off border rounded-lg p-4 wrapper instead of the shared Card primitive (Spending.tsx:240). The ideal uses the same Card token as every other tile so radius/shadow/border stay consistent if the design system shifts.
  • Kept as-is: placement directly under cashflow, the tabular-nums mono total, and the muted eyebrow label all match the shipped tile and the rest of the Spending page.
Dark mode

Dark-mode token mapping

  • Page bg #0f172a; card bg #1e293b; border #334155. Cairn already drives these via the shadcn CSS variables, so the cards inherit automatically — no per-component overrides needed.
  • The aggregate card's left accent rule should flip from #1e293b (near-invisible on dark) to a lighter slate #94a3b8 or the accent blue so it stays visible.
  • Person/status pill fills (the 10–12% color washes) read fine on dark, but the text in tag-active / tag-soon should lighten (e.g. #4ade80 / #fbbf24) for contrast.
  • The dashed "Add" ghost card keeps the accent border + accent text; only the hover wash needs a touch more opacity on dark.
